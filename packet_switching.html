Packet Switching — Cara Internet Mengirim Datamu

Setiap kali kamu kirim pesan, nonton YouTube, atau main game online — semuanya berkat packet switching!

Packet switching adalah cara mengirim data di jaringan dengan memecah data menjadi potongan-potongan kecil yang disebut paket. Setiap paket dikirim secara terpisah, bisa lewat jalur berbeda, lalu disusun kembali di tujuan.

Analogi: Kirim buku lewat paket pos

Daripada kirim satu buku besar, kamu robek tiap bab, masukkan ke amplop terpisah, dan kirimkan. Tiap amplop bisa lewat rute berbeda. Di tempat teman, semua bab dikumpulkan dan disusun ulang menjadi buku utuh lagi!

Analogi: Tol yang macet

Saat tol macet, kendaraan bisa lewat jalur lain. Paket data pun begitu — jika satu jalur penuh atau rusak, paket otomatis cari jalur lain tanpa perlu restart koneksi!

Tahukah kamu? Packet switching dikembangkan Paul Baran (Amerika, 1960-an) untuk militer AS — agar komunikasi tetap berjalan meski jaringan sebagian hancur. Teknologi ini kemudian jadi fondasi internet yang kita pakai hari ini!

Konsep kunci
▢

Paket

Potongan kecil data asli. Ukurannya ditentukan protokol jaringan, misalnya maks 1024 byte.

△

Router / Node

Perangkat yang meneruskan paket ke tujuan berikutnya — bisa router, switch, atau komputer.

↻

Routing

Proses pemilihan jalur terbaik. Router memilih jalur berdasarkan kepadatan lalu lintas saat itu.

■

Reassembly

Menyusun kembali paket-paket yang tiba di tujuan sesuai nomor urutnya menjadi data lengkap.

Isi sebuah paket data

Setiap paket terdiri dari tiga bagian. Bayangkan seperti amplop surat — ada bagian depan (alamat), isi surat, dan bagian belakang (tanda terima).

Header
• Alamat IP pengirim
• Alamat IP penerima
• Nomor urut paket
• Ukuran paket
Payload
• Data aktual
• Teks, gambar, video
• Maks ~64 KiB
Trailer
• Penanda akhir paket
• Kode cek error (CRC)
• Memastikan data utuh

Analogi: Amplop surat

Header = bagian depan amplop (nama & alamat, nomor amplop ke-3 dari 7)
Payload = isi surat di dalam amplop
Trailer = stiker "fragile" (bukti paket sampai utuh)

Apa itu CRC? Cyclic Redundancy Check adalah cara matematis mengecek apakah data rusak. Seperti barcode produk di supermarket — kalau kode tidak cocok, berarti ada yang salah!

Urutan proses lengkap
1

Pecah data menjadi paket

Data asli dipotong menjadi beberapa chunk. File 1 MB bisa dipecah menjadi ratusan paket kecil.

2

Tambahkan header dan trailer

Setiap paket diberi alamat IP pengirim, penerima, nomor urut, dan kode cek error.

3

Catat nomor urut

Jika data dipecah menjadi banyak paket, urutan tiap paket dicatat agar bisa disusun kembali.

4

Kirim melalui jaringan (Store & Forward)

Paket berpindah dari router ke router. Di tiap router, paket disimpan dulu, dicek, lalu diteruskan.

5

Susun ulang di tujuan

Setelah semua paket tiba, komputer penerima menyusun kembali paket sesuai nomor urut.

6

Kirim konfirmasi (ACK)

Penerima mengirim pesan konfirmasi ke pengirim. Jika tidak ada ACK, paket dikirim ulang otomatis.

Mengapa paket bisa hilang? Kadang paket mengalami bouncing data — berpindah-pindah antar router tanpa sampai ke tujuan. Solusinya adalah TTL! Baca tab "Hopping & TTL".

Hopping dan TTL (Time to Live)

Setiap paket diberi TTL — penghitung mundur untuk mencegah paket tersesat selamanya di jaringan.

Analogi: Tiket pesawat dengan batas transit

Tiket yang hanya boleh transit maksimal 10 kali. Setiap transit, hitungan berkurang satu. Kalau sudah 10 kali transit tapi belum sampai tujuan, tiket hangus dan harus pesan ulang. TTL bekerja persis seperti itu!

1

Paket diberi TTL maksimum

Saat pertama dibuat, paket diberi nilai TTL awal (misalnya 64 atau 128).

2

TTL berkurang di setiap router

Setiap kali paket meninggalkan sebuah router, nilai TTL dikurangi 1.

3

TTL = 0 → paket dihapus

Jika TTL sudah 0 tapi paket belum sampai, router menghapus paket tersebut dari jaringan.

4

Permintaan kirim ulang

Komputer penerima mendeteksi paket hilang dan meminta pengirim untuk mengirimkan ulang.

Fun fact: Buka Command Prompt dan ketik ping google.com. Di hasilnya ada "TTL=xxx" — itu sisa TTL dari server Google saat paket balasan sampai ke komputermu!

Dua jenis packet switching
1. Connectionless (Datagram Switching)

Setiap paket bebas memilih jalurnya

Setiap paket diperlakukan sebagai unit independen. Tidak ada jalur yang ditetapkan. Setiap router memutuskan ke mana paket harus dikirim berdasarkan kondisi jaringan saat itu.

Analogi: Ojek online tanpa rute tetap

Setiap paket seperti penumpang ojek yang dipesan sendiri-sendiri. Pengemudi bebas pilih jalan tercepat. Hasilnya, penumpang bisa tiba tidak berurutan!

Contoh: IP, Ethernet, UDP — digunakan internet kita sehari-hari.

2. Connection-oriented (Virtual Circuit)

Jalur sudah ditetapkan sebelum data dikirim

Jalur virtual dibentuk dulu dari pengirim ke penerima. Semua paket melewati jalur yang sama dan tiba berurutan.

Analogi: Kereta dengan jalur tetap

Jalur rel sudah ditetapkan sebelum kereta berangkat. Semua gerbong (paket) mengikuti jalur yang sama dan tiba berurutan.

Contoh: ATM, Frame Relay

Packet switching vs Circuit switching
Circuit switching

Jalur dedikasi

  • Jalur khusus dibentuk dulu
  • Tidak bisa dipakai pengguna lain
  • Boros bandwidth
  • Jika putus, gagal total
  • Bayar per durasi koneksi
  • Contoh: telepon analog
Packet switching

Jalur bersama

  • Tidak perlu jalur khusus
  • Jalur dipakai banyak pengguna
  • Hemat bandwidth
  • Rusak → rerouting otomatis
  • Bayar per data terkirim
  • Contoh: internet, chat, streaming

Menarik! Saat kamu telepon lewat HP (4G/5G), suaramu sudah pakai packet switching (VoIP) — bukan circuit switching seperti telepon kabel zaman dulu. Makanya kadang suara bisa putus-putus kalau sinyal lemah!

Packet switching dalam kehidupan sehari-hari

WhatsApp / Chat

Pesanmu dipecah jadi paket, dikirim lewat server, disusun kembali di HP temanmu. Semua dalam milidetik!

YouTube / Netflix

Video dipecah ribuan paket. Buffering terjadi karena paket datang tidak merata saat koneksi lambat.

Game Online

Posisi karakter dan aksi dikirim sebagai paket. Lag terjadi karena paket terlambat sampai ke server.

Kirim Foto / File

Foto 5 MB dipecah ribuan paket, dikirim bersamaan lewat berbagai jalur, disusun ulang di tujuan.

Video Call

Suara dan video dikirim sebagai paket real-time. Paket terlambat = suara robot atau video patah-patah.

Browsing Web

HTML, gambar, dan script dikirim sebagai paket terpisah — makanya halaman web bisa muncul bertahap.

Kelebihan

  • Tidak butuh jalur khusus yang boros
  • Kegagalan rendah karena bisa rerouting otomatis
  • Banyak pengguna bisa pakai jaringan yang sama
  • Mendukung kecepatan tinggi dan biaya lebih murah

Kekurangan

  • Paket bisa hilang dan harus dikirim ulang
  • Proses menyusun ulang butuh waktu tambahan
  • Delay bervariasi — kurang ideal untuk voice call berkualitas tinggi
